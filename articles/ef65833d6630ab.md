# はじめに

3Dプリンターを購入したソフトウェアエンジニアの方々に聞いてみたいんですが、「ちょっとした小物を作りたいけど、CADがまあまあめんどくさい」って思ったことありませんか？ありますよね？自分はあります。

最初は普通にCADも試してみました。やりたい気持ちはあるんですが、寸法を入れて、形を作って、細かいところを直して、STLにして……とやっていると、思ったより時間がかかります。ちょっとしたスタンドや治具を作るだけでも、かなり腰が重くなります。

Blender をAIから操作できる、みたいな話もあります。ただ、自分はそもそも Blender の操作がよくわかっていないので、「AIで操作できる」と言われても、まだめんどくささが勝ちます。

そこで「Pythonで3Dモデルを作れないのかな」と試しに調べてみたら、build123d というライブラリがあって、普通にできそうでした。

で、Claude Code と組み合わせて試してみたら、これがすごくよかったです。**Claude Code + build123d で3Dプリント用モデル作りがむっちゃ気軽になった**ので、紹介しますー。（もちろん、codex や他のAI Coding Agentでもできます！）

# やったこと

流れはこれだけです。

1. Claude Codeに「測量野帳を3冊立てられるスタンドをbuild123dで作って」と伝える
2. Pythonコードが生成される
3. STLを書き出す
4. macOSのプレビューで見る
5. 気になるところを自然言語で直してもらう

3Dプリントの基本フローとしては普通にこれです。

```text
3Dモデルを作る -> STLに出力する -> スライサーにかける -> プリントする
```

今回は、この「3Dモデルを作る」の部分を build123d の Python コードにしました。

https://build123d.readthedocs.io/

セットアップも軽いです。

```bash
uv init --no-readme
uv add build123d
```

# できたコード

全部を手で書いたというより、Claude Codeに作ってもらって、プレビューを見ながら「ここをこうしたい」と直していった感じです。

コード全体は長いので折りたたみます。

:::details notebook_stand.py

```python
from build123d import *
import subprocess, math

slot_w    = 10
wall      = 2
n_slots   = 3
height    = 25
flange_h  = 2
chamfer_s = 0.5

slot_d  = height
total_w = wall + (slot_w + wall) * n_slots  # 38mm
body_l  = 34

tri_w = 15  # 三角支えの横幅
tri_h = 15  # 三角支えの高さ
cut   = tri_w / 3  # 垂直カットの位置（底辺の1/3）

groove_w     = 1.0
groove_depth = 0.5
n_grooves    = 4

# 斜め面の情報
# 斜め面: (bx, tri_h) -> (bx + side*(tri_w-cut), tri_h/3)
# dx = tri_w-cut = 10, dz = -(tri_h - tri_h/3) = -10 -> 45 deg
face_len = math.sqrt((tri_w - cut)**2 + (tri_h - tri_h / 3)**2)

with BuildPart() as stand:
    # 台座
    with Locations((0, 0, flange_h / 2)):
        Box(total_w, body_l, flange_h)

    # 本体
    with Locations((0, 0, flange_h + height / 2)):
        Box(total_w, body_l, height)

    # 左右の三角支え
    for side in [-1, 1]:
        bx = side * total_w / 2
        with BuildSketch(Plane.XZ):
            with BuildLine():
                Polyline([
                    (bx,                        0),
                    (bx + side * (tri_w - cut), 0),
                    (bx + side * (tri_w - cut), tri_h / 3),
                    (bx,                        tri_h),
                    (bx,                        0),
                ])
            make_face()
        extrude(amount=body_l / 2, both=True)

    # 野帳スロット
    for i in range(n_slots):
        x = -total_w / 2 + wall + slot_w / 2 + i * (slot_w + wall)
        with Locations((x, 0, flange_h + height / 2)):
            Box(slot_w, body_l + 10, slot_d, mode=Mode.SUBTRACT)

    slit_depth = 1.0

    # 本体側面の縦スリット
    z_body_slit_start = tri_h
    z_body_slit_span  = (flange_h + height) - z_body_slit_start
    for i in range(n_grooves):
        y = -body_l / 2 + body_l / (n_grooves + 1) * (i + 1)
        for side in [-1, 1]:
            x = side * (total_w / 2 - groove_depth / 2)
            with Locations((x, y, z_body_slit_start + z_body_slit_span / 2)):
                Box(groove_depth + 0.1, groove_w, z_body_slit_span + 0.1, mode=Mode.SUBTRACT)

    # 三角支えの斜め面スリット
    for i in range(n_grooves):
        y = -body_l / 2 + body_l / (n_grooves + 1) * (i + 1)
        for side in [-1, 1]:
            bx = side * total_w / 2
            x_c = bx + side * (tri_w - cut) / 2
            z_c = (tri_h + tri_h / 3) / 2
            x_c += side * slit_depth / 2 / math.sqrt(2)
            z_c += slit_depth / 2 / math.sqrt(2)
            rot_y = side * 135
            loc = Location((x_c, y, z_c), (0, rot_y, 0))
            with Locations(loc):
                Box(slit_depth * 2, groove_w, face_len * 1.1, mode=Mode.SUBTRACT)

    # 三角支えの垂直端壁スリット
    wall_h = tri_h / 3
    for i in range(n_grooves):
        y = -body_l / 2 + body_l / (n_grooves + 1) * (i + 1)
        for side in [-1, 1]:
            bx = side * total_w / 2
            x_wall = bx + side * (tri_w - cut)
            x_c = x_wall - side * slit_depth / 2 / 2
            with Locations((x_c, y, wall_h / 2)):
                Box(slit_depth / 2, groove_w, wall_h, mode=Mode.SUBTRACT)

    # 上部エッジの面取り
    max_z = flange_h + height
    top_edges = [e for e in stand.edges() if abs(e.center().Z - max_z) < 0.1]
    chamfer(top_edges, chamfer_s)

export_stl(stand.part, "notebook_stand.stl")
print(f"生成完了: {total_w + (tri_w-cut)*2}mm x {body_l}mm x {flange_h + height}mm")
subprocess.Popen(["open", "notebook_stand.stl"])
```

# 何がよかったか

## 寸法の指示がゆるくていい

CADだと、自分の頭の中である程度ちゃんと形を決めてから操作する感覚があります。

でも今回は「スロットは10mmくらい」「三角支えは外側に少し出す」「高さは25mmくらい」みたいな雑な言い方で進められました。

もちろん最終的には寸法を詰める必要があります。ただ、最初の形を出すまでがかなり楽です。明確な完成イメージがなくても、まず形にして、そこから調整できます。

## 見た目も相談しながら作れる

機能だけなら、ただの四角いブロックにスロットを開ければ終わりです。

ただ、机に置くものなので、少し見た目も整えたくなります。今回は「ジャパンディっぽく」「縦スリットを入れたい」くらいの指示で、側面と三角支えにスリットを入れる方向になりました。

ジャパンディは北欧デザインと日本デザインを融合させたスタイルらしいです。3Dモデルを見ているとよく出てくるので、はやってるんですかね？おしゃれなので「ジャパンディっぽく」って言えばおしゃれになるはずです。知らんけど。

このへんは、ちゃんとしたデザイン指示というより、会話しながら方向性を探る感じです。これがAI Coding AgentとコードベースCADのいいところだと思います。

## CADなしで回せる

Blender や 3D CAD と AI を連携させるのも面白そうです。ただ、自分の場合はその前に「Blender の操作を理解する」「CADの作法を思い出す」という段階で止まりがちでした。

今回のような小物であれば、

```text
Claude CodeでPythonを書く -> STL出力 -> macOSのプレビューで見る
```

だけで十分回せました。

コードを直して実行すると、すぐSTLが開く。この軽さがかなりよかったです。

# ハマったこと

## スリットは意外と難しい

本体側面に縦スリットを入れるだけなら簡単です。

ただ、今回は左右の三角支えにもスリットを入れたかったので、斜め面に対して箱を回転させて差し引く必要がありました。

```python
loc = Location((x_c, y, z_c), (0, rot_y, 0))
with Locations(loc):
    Box(slit_depth * 2, groove_w, face_len * 1.1, mode=Mode.SUBTRACT)
```

このあたりは、座標と回転を少しずつ直しながら進めました。プレビューで見て「削れていない」「削れすぎている」を確認し、Claude Codeにそのまま伝える感じです。ただ、うまく伝わらず、こっちで修正する部分もありました。

## filletよりchamferのほうが安定した

最初は角を丸めたくて `fillet` を試しました。

ただ、壁が薄い箇所やスリットが近い箇所では失敗しやすかったです。今回は上部のエッジだけ軽く `chamfer` する形に落ち着きました。filletやchamferが何か今もわかってませんが、Claudeくんが試してくれてました。

```python
top_edges = [e for e in stand.edges() if abs(e.center().Z - max_z) < 0.1]
chamfer(top_edges, chamfer_s)
```

3Dプリントする小物なら、まずは面取りくらいで十分なことも多そうです。
