# はじめに

普段ターミナルで仕事しているエンジニアの方々に聞いてみたいんですが、「電子工作やりたいけど、なんかめんどくさい」って思ったことありませんか？ありますよね？・・・え？・・・・・・ない？？？

それが今、**Claude Code + arduino-cli でむっちゃ変わった**ので、紹介したいと思いますー。（もちろん、codex や他のAI Coding Agentでもできます！）

# 作ったもの

引き出しに眠っていたM5StickC Plusを引っ張り出して、こんなものを作ってみました。

**あんしんでんしょなお** — ボタンを押すと、家族のスマホに「ただいま」通知が届くデバイスです。

https://github.com/optimisuke/anshin-densho-nao

![](/images/2026-05-08-00-11-23.png)

子どもが帰宅したときにボタンを押すだけ。ntfy.sh（HTTPリクエストでプッシュ通知を送れるサービスです）経由でスマホに通知が届きます。シンプルですが、実用的でいい感じです。

# やったこと

1. Claude Codeに「M5StickCでボタンを押したらntfy.shにPOSTするやつ作って」と伝える
2. コードが生成される
3. そのままarduino-cliでコンパイル・書き込み

ということで、これだけです。

arduino-cliはArduino公式のCLIツールで、コンパイルから書き込みまでターミナルで完結しますー。

```bash
# ポート確認
arduino-cli board list

# コンパイル
arduino-cli compile --fqbn m5stack:esp32:m5stack_stickc_plus src

# 書き込み
arduino-cli upload --fqbn m5stack:esp32:m5stack_stickc_plus \
  --port /dev/cu.usbserial-XXXXXXXX src
```

`board list`で接続中のボードとポートを確認して、`compile`でビルドして、`upload`で書き込む、という流れです。`--fqbn`はボードの識別子で、M5StickC Plusの場合は`m5stack:esp32:m5stack_stickc_plus`になります。まぁ、これも覚えなくて大丈夫です。Claude Codeが全部実行してくれますー。

**Claude CodeがC++を書いて、arduino-cliがそのまま動かしてくれます**。IDEを開かなくていい。コードをコピペしなくていい。ターミナルから離れなくていい。むっちゃ良くないですか？

# なぜ今なのか

以前の電子工作の流れはこんな感じでした：

- サンプルコードを探す
- やりたいことを書く
- 動かない
- エラーをググる
- ライブラリのドキュメントを読む
- また詰まる

これが今は：

- やりたいことを自然言語で伝える
- 動くコードが出てくる
- arduino-cliで書き込む

詰まるポイントがほんとに減ったなと思ってます。もちろんハードウェア固有の問題や環境依存のエラーはまだあります。ただ、「C++わからん」「どのライブラリ使えばいい」「エラーの意味がわからん」あたりはAIが引き受けてくれるので、劇的に楽になった感じです。
