# はじめに

![ロボットアームが動く様子](/images/2026-05-08-robotarm.gif)

1億年前に買ったmicro:bitと、3Dプリントしたアームで動くロボットアームです。

電子工作デバイスとか3Dプリンターとか、買ったはいいものの途中で積んじゃうの、自分だけじゃないと思うんですが。「積んでるなぁ」というモヤモヤがずっとあったんですが、この春やっと成仏できました。3Dプリンターを買い直したことをきっかけに、Claude Codeと組み合わせたらむっちゃ捗ることに気づいたのが大きかったです。何が変わったのか整理しておきます。

# 積んできたもの

## Arduino — gitで管理したいのに、なんか微妙

Arduinoって触ってみるとセンサーのデータが取れたりLEDが光ったりして楽しいんですが、なかなか続かなかったんですよね。自分も何度か試して、そのたびに積んでました。

理由の一つがコード管理のしにくさかなと思います。コードはgitで管理したいんですが、Arduino IDEが前提のワークフローと噛み合わない。書き込みも毎回GUIでポチポチで、なんとなく自動化できない感じがずっとありました。「なんか微妙」で積む、を繰り返してました。

## CAD — 覚えるまでが長すぎる

3Dプリンターを手に入れてから、自分で設計したものを印刷したくなりました。Fusion 360等を試して、チュートリアルもやったんですが、「やりたいものを作れるようになるまで」が遠くて、結局既製モデルを印刷するだけで終わりました。

CADそのものが悪いわけじゃなくて、「この形を作りたい」→「CADで表現できる」までの距離が自分にはしんどかったです。

## 3Dプリンター — 失敗が積み重なる

1億年前に一度3Dプリンターを買って、しばらく遊んでました。

https://qiita.com/optimisuke/items/463958da24b0fb22f10f

反ったり途中で剥がれたり、設定を調べて印刷してまた失敗して、を繰り返してました。その後、ある程度満足してからはたまに触る程度になって、他のものが楽しくなってブランクができました。気づいたら部屋のすみに眠ってました。

この春、また買い直しました。Bambu Lab のA1 Mini です。むちゃくちゃ評判良いし今度こそちゃんと使いたいと思って。

# 何が変わったか

Claude Code等のAIコーディングエージェントを仕事で使い始めて、ちょっと電子工作にも使えそうだと思って試してみたら、むっちゃ良かったです。

## Arduino: arduino-cli + Claude Codeで全部CLIに

`arduino-cli`というツールがあります。ArduinoのボードへのコンパイルとアップロードをCLIからできます。これをClaude Codeから操作するようにしたら、ワークフローがガラッと変わりました。

コードはふつうのエディタで書いてスムーズにgit管理できます。書き込みもClaude Codeに「書き込んで」と言えば終わります。最初に試したとき、`git add .`が普通に使えたのがなんか嬉しかったです。「なんか微妙」の正体がGUIへの依存だったと気づきました。

## CAD: build123dをClaude Codeに書かせる

`build123d`というPythonベースの3Dモデリングライブラリがあります。「幅24mm、奥行き15mm、高さ20mmの箱を作って、底面中央にケーブルが通る10mmの穴をあけて」とClaude Codeに伝えると、コードを書いてくれてSTLが出てきます。

CADソフトを覚えなくても、形を言葉で説明できれば3Dモデルが作れます。「次は側面にM5をはめこむための爪を追加して」と続ければコードを修正してまた出してくれます。これは思ったより実用的でした。

## 3Dプリンター: 修正が速いと、失敗してもすぐ次を試せる

失敗したとき、以前はCADを開いて修正して書き出してという手順が必要でした。build123dにしてからは、コードの数値を変えて再生成するだけです。

さらに、Bambu Lab A1 Miniはほとんど失敗しないのがありがたいです。以前の3Dプリンターは反ったり剥がれたりが多かったんですが、A1 Miniはそういうストレスがほぼない。設計を直してすぐ印刷、がサクサク回せます。「ちょっと緩い」→ 数値を1mm変える → 再印刷、を何回でも気軽に試せるようになりました。

# あんしんでんしょなお2

調子に乗って、もう一個作りました。

以前M5StickCで「あんしんでんしょなお」というデバイスを作りました。ボタンを押すと家族のスマホに通知が届くだけのシンプルなもの。今回はM5Atomで2号機を作りました。M5Atomは切手くらいのサイズの正方形のデバイスで、ボタンが一つとLEDマトリクスがついてます。小さいので、壁への設置も簡単でした。

まず紙に「こんな形かな」とスケッチしました。

![設計スケッチ](/images/c9a2e4f7b831d5/sketch.jpg)

このスケッチをClaude Codeに見せて、build123dのコードを書いてもらいました。何度か寸法を調整して、印刷したのがこれです。

![3Dプリントしたホルダー単体](/images/c9a2e4f7b831d5/holder.jpg)

M5Atomをはめて壁に設置しました。

![壁に設置した様子](/images/c9a2e4f7b831d5/installed-close.jpg)

![設置位置の全体像](/images/c9a2e4f7b831d5/installed-wide.jpg)

コードもClaude Codeに書いてもらいました。Deep Sleep 等の機能がいまいちわかってなかったのですが、一瞬で実装してくれました。ライブラリの使い方調べたりしていた過去の悩みはなんだったのか。

# おわりに

道具は揃ってたのに、続ける気力がもたなかった。積みデバイスの原因ってだいたいそれな気がします。

覚えるコスト、管理のめんどくささ、失敗の立ち直りの遅さ、が積み重なって毎回途中で止まってました。Claude Codeで変わったのは、その一個一個のコストが下がったことです。これからはもっと気軽に電子工作できそうです。

積んでるデバイスや道具がある人は、ぜひ一回試してみてほしいです。成仏させましょう。
