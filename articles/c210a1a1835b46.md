## 前提

この記事は「既存のタスク管理がしっくりこない」「最小構成で自分用アプリを今日中に作りたい」人向けやで。
**PWA + localStorage + GitHub Pages** だけで、ログインなし・サーバなしの自分用タスク管理が動くようになる。

## はじめに

AI Coding Agent がある今なら、[PWA (Progressive Web Apps)](https://developer.mozilla.org/ja/docs/Web/Progressive_web_apps) 使ったら、あんたの好みに合わせてサクッと自作できるで。
デプロイや運用がしんどいのんはわかるけど、タスク管理アプリくらいならバックエンド作らずにフロントエンドだけで動くようにして [GitHub Pages](https://docs.github.com/ja/pages/getting-started-with-github-pages/what-is-github-pages) で公開して使ったらむっちゃラクやで。
自分の気に入った機能だけ作ったらええし、自分用ならログインも不要やしな。
しかも AI は Tailwind を理解してるから、見た目もええ感じに仕上げやすいねんで。まさに追い風やでぇ！

## じゃあどうすんねん

**PWA + localStorage** で、フロントエンドだけで完結したらむっちゃええで。
これで「アプリ感」と「保存」はちゃんとできるで。
**GitHub Pages**で公開できるしな。
React とか使わんでも、それっぽいの AI 使えば作れるで。

## 実現方法

やることはこの 3 ファイルだけやで。AI に頼めば作ってくれるで。

- `index.html`
  UI とロジックを 1 枚にまとめる。タスクの追加/更新/表示をここで完結。
- `manifest.webmanifest`
  アプリアイコンや表示名を定義して、PWA として“アプリっぽく”見せる。
- `sw.js`
  主要ファイルをキャッシュしてオフラインでも動くようにする。

### コアコード（抜粋）

中身はこんな感じやけど、正直わからんくても AI に頼めばいけるで。

`index.html`（UI・保存）

```html
<script>
  const STORAGE_KEY = "bokabikun.tasks.v1";
  const loadTasks = () => JSON.parse(localStorage.getItem(STORAGE_KEY) || "[]");
  const saveTasks = (tasks) =>
    localStorage.setItem(STORAGE_KEY, JSON.stringify(tasks));

  function markDone(id) {
    const tasks = loadTasks();
    const t = tasks.find((x) => x.id === id);
    if (!t) return;
    t.lastDone = new Date().toISOString().slice(0, 10);
    t.updatedAt = new Date().toISOString();
    saveTasks(tasks);
  }
</script>
```

アイコンも AI がつくってくれるで。
`manifest.webmanifest`

```json
{
  "name": "bokabikun",
  "short_name": "bokabikun",
  "start_url": ".",
  "display": "standalone",
  "background_color": "#0f172a",
  "theme_color": "#0f172a",
  "icons": [{ "src": "icon.svg", "sizes": "512x512", "type": "image/svg+xml" }]
}
```

`sw.js`（事前キャッシュ）

```js
const CACHE_NAME = "bokabikun-v1";
const PRECACHE_URLS = [
  "./",
  "./index.html",
  "./manifest.webmanifest",
  "./icon.svg",
];

self.addEventListener("install", (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME).then((cache) => cache.addAll(PRECACHE_URLS))
  );
});
```

### 公開方法

こことかみたらええで。もっとわかりやすいのもあるかもやけど。むっちゃ簡単やで。GitHub の設定画面で Pages で公開するみたいなことするだけやで。

https://docs.github.com/ja/pages/getting-started-with-github-pages/creating-a-github-pages-site

スマホのブラウザで開いて、「アプリとしてインストール」みたいなんしたらアプリっぽく動くで。

### AI に頼む時

「index.html だけでタスク管理アプリ作りたい。PWA にしたい。localStorage で保存したい。」って伝えたら勝手にやってくれるで。

## メリット

使ってみると、ここが気持ちええで。

- GitHub Pages でデプロイできて気楽やで
- バックエンドないからセキュリティ/プライバシーリスク気にしんくていいで
- ログインやサーバ運用が不要やで

## 注意点

もちろん弱点もあるで。

- 別端末への移行めんどいで
- ブラウザのデータ削除で消えるで

自分用ならエクスポートつけたりしたらどうにかなるで。
タスク管理アプリでも OpenAI の API 呼びたいとかあるんやったら、セキュリティ的に無理やから、公開はやめとき。

UI も気に入らんくなったら、AI に修正してもらったらええしな。
